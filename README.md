# 🧠 Memory Corruption — Puzzle Game

A minimal, experimental browser game where the interface itself becomes your biggest challenge.

As you play, the system begins to break:
- Controls stop behaving correctly
- Visuals distort and flicker
- Colors invert
- The game becomes harder to understand

Can you adapt before everything collapses?

---

## 🎮 Core Concept

This is not a traditional game.

Instead of increasing difficulty through enemies or levels, the **interface itself degrades over time**, forcing the player to rely on instinct, memory, and adaptability.

---

## ⚙️ Tech Stack

- HTML (structure)
- CSS (UI + glitch effects)
- Vanilla JavaScript (game logic)
- No frameworks
- No libraries
- No backend

---

## 🚀 Features

- 🧩 Simple puzzle objective (reach the goal)
- 🧠 Dynamic control corruption (keys randomly remap)
- 🎨 Progressive visual glitches:
  - Color inversion
  - Screen distortion
  - Flickering effects
  - Blur and instability
- ⏱ Increasing difficulty over time
- 💾 LocalStorage support for persistence (optional enhancements)
- 📱 Responsive design (works on desktop & mobile)

---

## 🕹️ Gameplay

1. Use arrow keys to move the player
2. Reach the target to win
3. Survive as:
   - Controls change unexpectedly
   - The screen becomes harder to read
   - The system becomes unstable

---

## 🧠 Core Mechanics

| System              | Behavior |
|--------------------|--------|
| Control Corruption | Key mappings randomly change |
| Visual Glitch      | CSS distortions and flickers |
| Color Inversion    | UI becomes disorienting |
| Time Pressure      | Corruption speed increases |
